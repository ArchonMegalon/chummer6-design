# Lost Potential Materialization Wave

**Product:** Chummer6 / SR Campaign OS  
**Status:** Active follow-on wave proposal  
**Purpose:** Convert the most compelling "living campaign" ideas into eight small, release-gated proofs that users can feel before broader simulation or social layers arrive.

## Product rule

Chummer6 should not become a generic living-world simulator.
It should become the trusted campaign memory and consequence engine for Shadowrun.

The first loop must stay simple:

```text
Create or adopt campaign
-> prep a run
-> play with action help
-> close the run
-> update runner goals
-> city reacts
```

## Gates

### Gate 1 — Live SR6 combat round

```text
A player and GM can complete one SR6 combat round with:
  initiative
  major/minor actions
  action conversion
  between-turn affordances
  mook damage
  condition changes
  explain receipts
```

References:

* `LIVE_ACTION_ECONOMY_AND_TURN_ASSIST.md`
* `GM_RUNBOARD_LIVE_OPERATIONS.md`

### Gate 2 — Local source page opens from explain drawer

```text
A user binds a local PDF.
A calculated value shows a source anchor.
Click opens the local PDF to the right page.
No PDF is uploaded.
No raw local path enters telemetry.
```

References:

* `SOURCE_ANCHOR_AND_LOCAL_RULEBOOK_BINDING.md`

### Gate 3 — Existing campaign adoption

```text
GM imports or enters existing runners.
Marks unknown history.
Adds house rules and active contacts.
Starts ledger from today.
Gets adoption confidence report.
```

References:

* `CAMPAIGN_ADOPTION_WIZARD.md`

### Gate 4 — Runner pins an upgrade

```text
Player pins desired cyberware, spell, focus, deck, drone, or vehicle.
Chummer shows nuyen, karma, downtime, and approval requirements.
Reward from a run updates progress.
```

References:

* `RUNNER_RESUME_AND_GOAL_PINS.md`

### Gate 5 — GM creates a prep packet

```text
GM generates or drafts:
  hook
  patron
  target
  legwork board
  opposition packet
  complication
  reward model
  player-safe brief
```

References:

* `PREP_PACKET_FACTORY_AND_PROCEDURAL_TABLES.md`

### Gate 6 — GM closes a run

```text
GM files ResolutionReport.
Chummer proposes consequences.
GM approves or edits.
Campaign state updates.
```

References:

* `BLACK_LEDGER_MVP_001.md`

### Gate 7 — BLACK LEDGER talks back

```text
ResolutionReport creates WorldTick.
WorldTick creates one player-safe NewsItem.
Next-session cockpit shows the consequence.
```

References:

* `BLACK_LEDGER_MVP_001.md`
* `NEWSREEL_AND_CITY_TICKER_MODEL.md`

### Gate 8 — Safe seasonal recognition

```text
Approved run resolution emits typed ReputationEvents.
Private or campaign-scoped honors update.
No global ladder.
No public shame.
No Table Pulse scoring.
```

References:

* `NETWORK_REPUTATION_AND_LEADERBOARDS_MODEL.md`
